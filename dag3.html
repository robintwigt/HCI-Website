Dag 3 - woensdag 5 juni

De dag begon met een presentatie van Joost. De presentatie ging over sociale robots: Robots die denken en voelen. Ik vond de presentatie niet erg relevant voor mij. Het ging namelijk vooral over het onderwerp ‘Artificial Creatures’.

Daarna was het weer tijd voor de daily standup. We hebben besproken wat er vandaag allemaal gedaan kan worden en zijn vervolgens gaan experimenteren met de Makey Makey. Het is handig als we weten hoe dit werkt, want misschien kunnen we het toepassen in ons concept. 

Na een korte brainstormsessie over ons concept, kwamen we erachter dat het handig is om ook gebruik te maken van een sensor die laat weten dat je naar voren moet lopen. Hiervoor moeten we dus een derde button toevoegen en een derde sensor. 

Ik heb de code geschreven die gebruik maakt van 3 buttons en 3 LEDs. Als je button 1 indrukt, gaat LED 1 branden en hetzelfde gebeurt bij 2 en 3. Hierdoor kunnen we dus gebruik maken van 3 sensoren die trillen. De code die ik heb geschreven is hieronder te lezen. De code had een stuk korter gekund, alleen ik wilde er ook voor zorgen dat als je meerdere buttons indrukt dat er dan ook meerdere LEDs gaan branden. Het resultaat is daaronder te zien in de vorm van een video.

// constants won't change. They're used here to set pin numbers:
const int buttonPin1 = 1; // the number of the pushbutton pin
const int buttonPin2 = 2; // the number of the pushbutton pin
const int buttonPin3 = 3; // the number of the pushbutton pin
const int ledPin1 = 13; // the number of the LED pin
const int ledPin2 = 12; // the number of the LED pin
const int ledPin3 = 11; // the number of the LED pin

void setup() {
// initialize the LED pin as an output:
pinMode(ledPin1, OUTPUT);
// initialize the LED pin as an output:
pinMode(ledPin2, OUTPUT);
// initialize the LED pin as an output:
pinMode(ledPin3, OUTPUT);
// initialize the pushbutton pin as an input:
pinMode(buttonPin1, INPUT);
// initialize the pushbutton pin as an input:
pinMode(buttonPin2, INPUT);
// initialize the pushbutton pin as an input:
pinMode(buttonPin3, INPUT);
}

void loop() {
// check if the pushbutton is pressed. If it is, the buttonState is HIGH:
if ((digitalRead(buttonPin1) == HIGH) && (digitalRead(buttonPin2) == HIGH) && (digitalRead(buttonPin3) == HIGH)) {
// turn LED on:
digitalWrite(ledPin1, HIGH);
digitalWrite(ledPin2, HIGH);
digitalWrite(ledPin3, HIGH);
}
else if ((digitalRead(buttonPin1) == HIGH) && (digitalRead(buttonPin2) == HIGH) && (digitalRead(buttonPin3) == LOW)) {
// turn LED on:
digitalWrite(ledPin1, HIGH);
digitalWrite(ledPin2, HIGH);
digitalWrite(ledPin3, LOW);
}
else if ((digitalRead(buttonPin1) == HIGH) && (digitalRead(buttonPin2) == LOW) && (digitalRead(buttonPin3) == HIGH)) {
// turn LED on:
digitalWrite(ledPin1, HIGH);
digitalWrite(ledPin2, LOW);
digitalWrite(ledPin3, HIGH);
}
else if ((digitalRead(buttonPin1) == LOW) && (digitalRead(buttonPin2) == HIGH) && (digitalRead(buttonPin3) == HIGH)) {
// turn LED on:
digitalWrite(ledPin1, LOW);
digitalWrite(ledPin2, HIGH);
digitalWrite(ledPin3, HIGH);
}
else if (digitalRead(buttonPin1) == HIGH) {
// turn LED on:
digitalWrite(ledPin1, HIGH);
digitalWrite(ledPin2, LOW);
digitalWrite(ledPin3, LOW);
}
else if (digitalRead(buttonPin2) == HIGH) {
// turn LED on:
digitalWrite(ledPin1, LOW);
digitalWrite(ledPin2, HIGH);
digitalWrite(ledPin3, LOW);
}
else if (digitalRead(buttonPin3) == HIGH) {
// turn LED on:
digitalWrite(ledPin1, LOW);
digitalWrite(ledPin2, LOW);
digitalWrite(ledPin3, HIGH);
}
else {
// turn LED off:
digitalWrite(ledPin1, LOW);
digitalWrite(ledPin2, LOW);
digitalWrite(ledPin3, LOW);
}
}

We hadden als idee om die 3 sensoren in een riem te doen. 1 sensor aan de voorkant en 2 sensoren aan de zijkant van de riem. Dit leek ons echter toch niet zo’n goed plan, omdat we dachten dat je dat waarschijnlijk niet goed gaat voelen. De riem zit namelijk over allemaal kleren heen en bovendien zal het ook nog eens erg moeilijk zijn om te voelen van welke kant de trilling komt.

We hebben dus besloten om toch gebruik te maken van 2 sensoren. Het leek ons alleen wel handig om ook een signaal te bedenken om te laten weten dat er vooruit gelopen moet worden. Dit gaan we doen door allebei de sensoren te laten trillen. 

Voor dit concept moest de code dus aangepast worden. Er moet namelijk 1 LED weg en de derde button moet er voor zorgen dat allebei de LEDs gaan branden. De code heb ik aangepast, waardoor het nu wel werkt zoals het moet. Hieronder is de aangepaste code te zien en het resultaat daarvan.

Daarna heb ik de schets van het concept aangepast. Ik heb 2 verschillende opties geschetst. Op de eerste schets is een arcade box te zien. Dit is de eerste optie om de bandjes mee te besturen. De tweede optie is te zien op de schets daaronder. Dit is een armband met de drie knoppen. Ook hiermee zou je de twee bandjes kunnen aansturen.

We weten nog niet voor welke besturing we gaan kiezen. De arcade box vinden we erg leuk, omdat het er dan echt als een spel uitziet. Een nadeel hiervan is dat we het dan draadloos moeten maken en we weten nog niet of dat gaat lukken. De armband vinden we erg leuk, omdat dit ook meteen een wearable is. Dit ziet er misschien iets minder spectaculair uit dan de arcade box, maar hierdoor hoeven we het niet draadloos te maken. Met deze armband kan je namelijk achter de persoon aanlopen.

De buttons van Arduino zijn erg klein en onhandig om te gebruiken. Vandaar dat we andere buttons willen gebruiken voor ons concept. We hebben besloten om arcade buttons te kopen, omdat die goed aansluiten bij het idee dat we nu in ons hoofd hebben.

De sensoren die kunnen trillen waren inmiddels bezorgd en dus konden we deze uittesten. Ik heb de trilsensoren vervangen voor de LEDs in mijn Arduino bord. We hadden 2 trilsensoren, maar 1 van de 2 sensoren deed het helaas niet.  In de video hieronder is te zien dat de trilsensor werkt als er op de button gedrukt wordt.

Uiteindelijk willen we ervoor kunnen zorgen dat de trilsensoren draadloos werken. Dit kan door middel van een Lilypad. We hebben dus wat informatie opgezocht over de Lilypad, omdat we er nog niet veel vanaf wisten. Toen kwamen we tot de conclusie dat we waarschijnlijk Lilypad trilsensoren nodig gaan hebben. Deze hebben we dan ook besteld.  Naast het informatie zoeken over de lilypad, zijn er ook een aantal dingen uitgeprobeerd. 

Aan het eind van de dag hebben we het scrum bord nog even bijgewerkt. De foto hiervan is hieronder te zien.
