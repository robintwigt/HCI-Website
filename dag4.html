Dag 4 - donderdag 6 juni

De dag begon zoals altijd met de daily standup. We bespraken weer wat we gisteren gedaan hadden en wat we vandaag wilden gaan doen.

Meteen na de daily standup begon de presentatie van Teun. Teun heeft een aantal van zijn projecten aan ons uitgelegd. Ik heb een aantal dingen opgeschreven tijdens de presentatie:

Electronic lifeforms and media in public spaces.
Ledwork: lichten die je aan elkaar kan koppelen en die op elkaar reageren.
Gesprek aangaan met dingen zoals een busstop .

We zijn daarna gestart met het schrijven van de promise. Iedereen schreef al zijn ideeën op in het Google Drive bestand. De promises die we bedacht hadden, zijn:

We promise to be your eyes
We promise to not hurt you
We promise to be your guide
We promise to not let you walk into a wall
We promise to maintain your trust (in us)
We promise to keep our promise
We promise you can walk without seeing
We promise that you can rely on your senses
We promise to give you a real life gaming experience
We promise you to experience what it’s like to be in a game
We promise you to feel like what it is to be controlled

Na het bedenken van alle promises, hebben we gestemd op de promise die wij het best vinden. Uiteindelijk was er het meest gestemd op de promise ‘We promise to be your guide’. Dit hebben we op een groot blad geschreven en opgehangen boven ons scrum bord.

Vervolgens ben ik het doolhof gaan schetsen die we nodig gaan hebben voor de expo.

Gisteren (dag 3) waren er al een aantal dingen uitgeprobeerd met de Lilypad, dit was echter niet helemaal gelukt. We zijn vandaag daarom verder gegaan met het testen van de Lilypad. Het lukte ons om ervoor te zorgen dat als je een button indrukt, dat er dan een LED gaat branden. Dit is te zien in de video hieronder.

Vervolgens wilden we ook kijken of dit werkt met de trilsensor. We hebben daarom de LED vervangen door de trilsensor. Gelukkig werkte dit ook, de trilsensor ging namelijk trillen wanneer je de button indrukte. De video hiervan is hieronder te zien.

Bij ons concept gaan we gebruik maken van 2 trilsensoren. Hierom hebben we een tweede LED toegevoegd en een tweede button. We wilden ervoor zorgen dat als je button 1 indrukt, dat dan LED 1 gaat branden en als je button 2 indrukt, dat dan LED 2 gaat branden. Ik had gisteren al een code hiervoor geschreven, deze code werkte wel met het Arduino bord, maar niet met de Lilypad. Hierom heb ik een andere code geschreven waarbij ik gebruik heb gemaakt van button states. De code die uiteindelijk werkte, is hieronder te lezen.

// Pin the led is attached to
int led1 = 9;
int led2 = 6;

// Pin the button is attached to
int buttonPin1 = 11;
int buttonPin2 = 10;

// Variable to store the button's state
int buttonState1 = 0;
int buttonState2 = 0;

void setup()
{
// Set the led pin as an OUTPUT
pinMode(led1, OUTPUT);
//Set the button as INPUT
pinMode(buttonPin1, INPUT_PULLUP);
// Set the led pin as an OUTPUT
pinMode(led2, OUTPUT);
//Set the button as INPUT
pinMode(buttonPin2, INPUT_PULLUP);
// Initialize Serial, set the baud rate to 9600.
Serial.begin(9600);
}

void loop()
{
buttonState1 = digitalRead(buttonPin1);
// Display button press in Serial Monitor
Serial.print("buttonState is:");
Serial.println(buttonState1);
// Print the buttonState
if (buttonState1 == LOW) {
digitalWrite(led1,HIGH);
//Display button press in Serial Monitor
Serial.println("Button is PRESSED");
} else if (buttonState1 == LOW) {
digitalWrite(led1,HIGH);
//Display button press in Serial Monitor
Serial.println("Button is PRESSED");
} else {
digitalWrite(led1,LOW);
//Display button press in Serial Monitor
Serial.println("Button is NOT PRESSED");
}

buttonState2 = digitalRead(buttonPin2);
// Display button press in Serial Monitor
Serial.print("buttonState is:");
Serial.println(buttonState2);
// Print the buttonState
if (buttonState2 == LOW) {
digitalWrite(led2,HIGH);
//Display button press in Serial Monitor
Serial.println("Button is PRESSED");
} else if (buttonState2 == LOW) {
digitalWrite(led2,HIGH);
//Display button press in Serial Monitor
Serial.println("Button is PRESSED");
} else {
digitalWrite(led2,LOW);
//Display button press in Serial Monitor
Serial.println("Button is NOT PRESSED");
}
}

Hierna moesten we alleen nog een derde button toevoegen. Deze button moest ervoor gaan zorgen dat beide LEDs gaan branden als je de button indrukt. Ook hierbij werkte mijn code van gisteren niet, dus moest ik hem herschrijven met button states. Uiteindelijk is het me gedeeltelijk gelukt. De LEDs doen namelijk wat ze moeten doen bij elke button, alleen knipperen ze. Ik heb de code een paar keer helemaal omgegooid, maar we konden er niet achter komen waar dit probleem aan lag. Toen hebben we maar besloten om het zo te laten. We denken namelijk dat het niet erg gaat zijn, omdat we gebruik gaan maken van trilsensoren. Het valt dan waarschijnlijk minder op dan bij de LEDs. Als we er later achter komen dat het toch erg storend is bij de trilsensoren, dan gaan we dit probleem nog proberen op te lossen. De code die gedeeltelijk werkte en het resultaat daarvan, is hieronder te zien.
